Setting up a modern PHP Project with great Workflow

Overview

This post is about setting up a new PHP project. When creating a new project, it is important to make sure that the workflow is easy to follow and can later be automated. See Continuous Integration

Summary

  • We are going to use Composer Autoload to manage our packages and dependencies. – Everytime a new class is added, autoload file will have to be regenerated.
  • Sample code contains a Cat class with two unit tests. – If a cat is fed, it will not be hungry. – If a cat is not fed, it will be hungry.
  • The unit tests are written using PHPUnit.

Technology

  • PHP
  • Install Composer
  • Install PHPUnit

For full installation guide, visit PHP, Composer and PHPUnit.

# Setting up PHPUnit
php composer.phar require phpunit/phpunit

Project Structure

  • app - contains the application – app/libraries - contains application libraries
  • test - contains project tests – phpunit.xml - configuration file for tests
  • composer.json - packages information – vendor - packages installed by composer
  • readme.md - $this

app/libraries/Cat.php

<?php

class Cat {
    private $name;
    private $isFed;

    /**
     * @param string $name - The name of the cat
     */
    public function __construct($name) {
        $this->name = $name;
    }

    /**
     * @param bool $hasBeenFed - indicates whether the cat has been fed
     */
    public function isFed($hasBeenFed) {
        $this->isFed = $hasBeenFed;
    }

    /**
     * The cat be hungry if it has not been fed
     */
    public function isHungry() {
        return !$this->isFed;
    }
}

test/CatTest.php

<?php

class CatTest extends PHPUnit_Framework_TestCase {
    
    public function testIsHungry_hasNotBeenFed_shouldBeHungry() {
        $momo = new Cat("Momo");
        $momo->isFed(false);
        $this->assertEquals(true, $momo->isHungry());
    }
    
    public function testIsHungry_hasBeenFed_shouldNotBeHungry() {
        $momo = new Cat("Momo");
        $momo->isFed(true);
        $this->assertEquals(false, $momo->isHungry());
    }
}

phpunit.xml

<phpunit bootstrap="vendor/autoload.php"
    colors="true"
    convertErrorsToExceptions="true"
    convertNoticesToExceptions="true"
    convertWarningsToExceptions="true"
    processIsolation="false"
    stopOnFailure="false"
    syntaxCheck="false">
    <testsuites>
        <testsuite name="App Tests">
            <directory>test</directory>
        </testsuite>
    </testsuites>
</phpunit>

composer.json

{
    "require": {
        "phpunit/phpunit": "^5.7"
    },
    "autoload": {
        "classmap": [
            "app/libraries"
        ]
    }
}

Using Composer Autoload

php composer.phar dump-autoload

Running Tests

php vendor/phpunit/phpunit/phpunit

An alias can be created to shorten the command and/or the libraries can be installed globally.

alias run-tests="php vendor/phpunit/phpunit/phpunit"
run-tests
